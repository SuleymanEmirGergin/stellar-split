# Pitch Script — Module 08

## Purpose

Generate two complete, word-for-word pitch scripts for the winning idea: a 60-second version (elevator/lightning round format) and a 3-minute version (standard judging session). Both scripts are immediately usable — no blank-filling required after this module.

---

## Pitch Architecture

Both scripts follow the same 7-part narrative arc. The 3-minute version expands each part; the 60-second version compresses.

```
1. Hook         → Grabs attention, creates tension
2. Problem      → Stakes, scale, urgency
3. Solution     → One sentence, then no more
4. Why Now      → The specific shift that makes this possible today
5. Demo Bridge  → Transition to or summary of the demo
6. Impact       → The metric that they'll remember
7. Ask          → Clear, specific, appropriately calibrated
```

---

## The 60-Second Pitch Script

**Total word count target: 130-150 words (people speak ~130 wpm)**
**Format: Continuous prose — no bullet points in delivery**

```
60-SECOND PITCH: [Product Name]
──────────────────────────────────────────────────────

[HOOK — 10-15 words]
"[Stat or observation that creates immediate tension about the problem domain]."

[PROBLEM — 25-30 words]
"[Specific user] [specific situation]. The current solution? [Current workaround in 1 sentence]. 
It costs [specific consequence — time, money, lives, opportunity]."

[SOLUTION — 15-20 words]
"[Product name] is a [product type] that [action verb] [outcome] for [user] — in [timeframe]."

[WHY NOW — 10-15 words]
"This is possible today because [specific technological or regulatory shift]."

[DEMO BRIDGE — 15-20 words]
"In our demo, you'll see [persona] [do action] and get [output] — in [X seconds]."

[IMPACT — 15-20 words]
"[X]% [metric improvement]. Across [scope], that's [larger impact]. 
[Product name] — [tagline]."

[ASK — 10-15 words]
"We're looking for [specific ask]. Thank you."

──────────────────────────────────────────────────────
Word count: [count] words (~[time] seconds at 130 wpm)
```

## The 3-Minute Pitch Script

**Total word count target: 380-420 words**

```
3-MINUTE PITCH: [Product Name]
──────────────────────────────────────────────────────

[HOOK — 20-30 words]
"[Story or scenario: a specific person, in a specific moment, with a specific problem. 
Create a scene. Make it vivid.]

Right now, this is happening to [N] [users] every [period]."

─────────────────

[PROBLEM — 60-80 words]
"Let me be specific about what's broken:

[Pain point 1] — [specific consequence].
[Pain point 2] — [specific consequence].
And here's the part that's easy to miss: [non-obvious related pain that goes deeper].

The tools that exist today? [Why existing solutions fail in 1-2 sentences].

This isn't a workflow problem. This is a [framing of the real root cause — systemic, structural, human]."

─────────────────

[SOLUTION — 50-70 words]
"[Product name] is [product type] that [action verb] [outcome].

Here's how it works in one sentence: [User does X] → [product does Y] → [user gets Z].

What makes it different from [closest alternative]:
[Differentiator 1 — specific].
[Differentiator 2 — specific].

And beneath that: [technical foundation — the deterministic core or specific AI mechanism]."

─────────────────

[WHY NOW — 30-40 words]
"This exists today, and not 2 years ago, because [specific shift — model capability / API availability / regulation / data access].

The window is [timeframe]. [Who else could move, and why we move first]."

─────────────────

[DEMO BRIDGE — 20-30 words]
"Let me show you [persona] — [role, situation, stakes].

Watch specifically what happens to [key element on screen] when [persona] [action].
[Presenter transitions to demo or describes what was just shown.]"

─────────────────

[IMPACT — 50-70 words]
"In our prototype / test / simulation:

[Primary metric]: [Specific number] vs. baseline of [current state].
[How we measured it]: [Brief methodology — makes it credible].
[Supportive signal]: [Secondary evidence — user interviews, existing analogues, market data].

Scaled to [market scope], this represents:
[Impact in dollars / lives / hours / decisions] per [period]."

─────────────────

[BUSINESS MODEL — 30-40 words]
"[Product name] charges [user type] [pricing model] — $[X] per [unit] per [period].

Our beachhead: [specific segment] — reachable via [channel].
Unit economics: [customer pays back in [X] [time period]].

This is a [SaaS / marketplace / usage-based] model at [~X]% gross margin."

─────────────────

[ASK — 25-35 words]
"We're [current stage]. We're looking for [specific ask: introductions / pilot customers / feedback / investment level].

If you know [type of person or organization we need], we'd love an introduction after this.

[Product name] — [tagline]. Thank you."

──────────────────────────────────────────────────────
Word count: [count] words (~[time] seconds at 130 wpm)
```

## Jury Calibration (from 16_judge-profiler.md)

After generating the base scripts, apply these adjustments based on jury type detected:

| Jury Type | Pitch Tone | Emphasize | De-emphasize |
|-----------|-----------|-----------|--------------|
| Investor | Crisp, confident, metric-first | Market size, moat, why now | Technical details |
| Domain Expert | Respectful of complexity, precise terminology | Domain accuracy, user research evidence | Market size |
| Technical | Direct, architecture-aware | Technical decision explained, eval method | Business model |
| Impact Optimist | Warm, human-centric | Persona's story, who benefits, equity | Unit economics |
| Pragmatist | Practical, integration-aware | Deployment path, existing system fit | Vision/scale |
| Mixed panel | Balanced across all | One element per archetype represented | Nothing |

## Pitch Anti-Patterns

```
❌ Starting with "My name is X, and our product is Y..."
   WHY: Name comes last. Judges don't vote for names. Lead with the problem.

❌ "Our solution uses cutting-edge AI to..."
   WHY: Every team says this. Name the SPECIFIC mechanism instead.

❌ The feature list
   WHY: "It does X, Y, Z, and also W" = forgettable. One core mechanism, explained well.

❌ "Our team has [10 combined years of] experience in..."
   WHY: If your demo is strong, experience is a bonus. If demo is weak, experience won't save it.

❌ Hedging the impact
   WHY: "This could potentially help up to..." = no conviction. State the number. Explain how.

❌ Running over time
   WHY: It signals poor preparation AND it literally prevents the last points from landing.
   FIX: Time every practice run. Cut until comfortably 10 seconds under.

❌ Building to the solution too slowly
   WHY: Judge attention peaks in first 30 seconds. Problem + solution must both be clear by then.
```

## The "Tagline" Rule

The pitch ends on a tagline. This is not optional.

A good tagline:
- Is 5-10 words maximum
- States what the product does for whom
- Does NOT include: "revolutionize", "disrupt", "change", "world"
- Can be understood by a 10-year-old
- Is memorable after hearing it once

**Tagline formula options:**
```
Option A (Action): "[Product] [verbs] [problem] for [user]"
Option B (Outcome): "Now [user] knows [insight they didn't have before]"
Option C (Before/After): "From [old state] to [new state]. In [time]."
Option D (Simple): "[Product] — [6-word description of the value]"
```
