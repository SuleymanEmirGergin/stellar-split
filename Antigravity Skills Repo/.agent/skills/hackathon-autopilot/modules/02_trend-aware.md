# Trend-Aware Mode — Module 02

## Purpose

Every winning hackathon idea should feel current — not timeless, not retro. Attach 2-3 trend hooks from the following categories to each idea, choosing only those that genuinely fit. Do not force-attach trends.

---

## 2026 Trend Categories

### AI / Agent Trends
- **Agentic workflows**: AI that acts, not just answers — executes multi-step plans autonomously
- **Multi-agent systems**: specialized agents that collaborate and hand off tasks
- **Computer use agents**: AI that operates software interfaces directly
- **Structured / deterministic AI**: JSON-output models with eval, not just chatbots
- **AI reasoning chains**: step-by-step transparent decision-making (CoT, ToT)
- **Real-time AI inference**: sub-second AI responses in user-facing products
- **Local / on-device LLMs**: privacy-first, offline-capable AI (Ollama, Core ML, TFLite)
- **LLM fine-tuning at scale**: LoRA, QLoRA, domain-adapted models
- **Multimodal AI**: image + text + audio + video processed together
- **AI safety and red-teaming**: bias detection, adversarial robustness, hallucination guards

### Data / Intelligence Trends
- **Real-time decision intelligence**: live data → instant recommendation or action
- **Synthetic data generation**: creating training/test data when real data is scarce
- **Federated learning**: model training across distributed private data sources
- **Graph intelligence**: knowledge graphs + graph neural networks for complex relationships
- **Time-series forecasting**: predictive analytics on temporal data streams
- **Data lineage & provenance**: tracking where data came from and how it was transformed

### Trust / Compliance Trends
- **Trustworthy AI**: explainability panels, confidence scores, audit trails
- **Privacy-first design**: data minimization, on-device processing, differential privacy
- **Audit logs & compliance-ready AI**: GDPR, AI Act, SOC 2 alignment visible in design
- **Human-in-the-loop**: AI surfaces the decision, human approves — accountability preserved
- **AI watermarking / provenance**: tracking AI-generated content origins

### UX / Interaction Trends
- **Invisible → visible**: surfacing hidden patterns users didn't know existed
- **Voice-first interfaces**: conversational UX beyond typing
- **Augmented reality overlays**: spatial computing, context-aware information
- **Accessibility-first AI**: voice, vision impairment, cognitive accessibility features
- **Emotion-aware interfaces**: adapting UX to user state (calm, stressed, urgent)

### Infrastructure / Edge Trends
- **Edge computing**: processing at the device/network edge, not the cloud
- **WebAssembly (WASM)**: near-native performance in the browser, offline-capable
- **Serverless + event-driven**: function-as-a-service patterns at scale
- **Open protocol adoption**: ActivityPub, AT Protocol, MCP — decentralized systems
- **AI infrastructure economics**: cost-per-inference optimization, model caching

### Domain-Specific Trends (2026)
- **Climate tech AI**: carbon tracking, energy optimization, climate risk modeling
- **Healthcare AI**: diagnostic support, care coordination, patient monitoring
- **Fintech AI**: fraud detection, credit scoring, regulatory compliance automation
- **Education AI**: personalized learning paths, skill gap analysis, tutoring agents
- **Supply chain resilience**: real-time disruption detection, alternative routing
- **Urban intelligence**: smart city sensors, mobility optimization, public service AI

---

## Application Rules

1. **Attach only genuinely relevant trends** — not every idea needs 3 trends from 3 categories
2. **Trend hook format**: "[Trend name]: [How this specific idea uses it — 1 sentence]"
3. **Avoid trend washing**: don't say "it uses AI" — say what the AI specifically does
4. **Judge / VC resonance**: prioritize trends that match the jury flavor from `16_judge-profiler.md`
5. **"Why now" narrative**: one trend per idea must explain why this is possible TODAY and not 2 years ago
