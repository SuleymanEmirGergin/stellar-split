# Social Amplifier — Module 29

## Purpose

Most hackathon teams focus 100% on building and ignore the **social visibility layer**. This is a mistake — judges follow hashtags, organizers retweet in-progress builds, and other participants are potential collaborators and future employers.

This module creates a **complete social media playbook** for the hackathon: before, during, and immediately after.

---

## Activation

Activate when:
- User asks about social media strategy
- User says "should we post about this?"
- User wants post templates
- `/social`

---

## Core Principle

**Authenticity > polish during a hackathon.** Raw, real-time builds outperform produced content every time. The constraint of the hackathon IS the content.

**Target audiences (by priority):**
1. Judges & organizers (they DO check social during the event)
2. Other hackathon participants (collaboration, future teammates)
3. Your professional network (career signals)
4. General tech community (longer-term visibility)

## Phase 1: Pre-Hackathon (24-48 hours before)

### LinkedIn — Team Announcement Post
```
TEMPLATE:
"[Excited / Thrilled / Pumped] to be joining [Hackathon Name] this weekend with [teammate tags or 'my team'].

We'll be working on [vague but intriguing description — don't reveal the full idea yet].

The challenge: [X hours] to build something that [solves / helps / transforms] [domain].

If you'll be there, let's connect! 👀

#[EventHashtag] #hackathon #[domain]tech #buildinpublic"

TIPS:
- Tag the event organizer's LinkedIn page
- Include a "who we are" line — makes it personal
- End with a question or CTA to drive engagement
- Do NOT reveal your full idea (competitive; also, it might change)
```

### Twitter/X — Team Thread Opener
```
TEMPLATE (Tweet 1 of thread):
"🏁 We're entering [Hackathon Name] in [N] hours.

Challenge: [X]h to build [domain] solution from scratch.

Team: [Your handle] + [teammate handles]

🧵 Following along? I'll be live-tweeting the build. [EventHashtag]"

WHY: Creates an audience before you even start.
```

## Phase 2: During the Hackathon (real-time build posts)

### Timing Cadence
- **Hour 1-2:** Setup post ("we're live")
- **Hour 4-6:** First working thing screenshot
- **Hour 8-10:** Progress update (what's working, what's hard)
- **Hour 16-20:** Pre-final sprint post (creates urgency + engagement)
- **Hour 22-24:** "We submitted" post

### Template Set — Build Updates

**"We're Live" (Hour 1):**
```
Twitter/X:
"⚡ We're live at [EventHashtag]. 

[H] hours, [N] people, and a problem we can't stop thinking about.

Here's what we're building... 🧵"

(Then in thread: 1 tweet describing the problem. No solution yet.)
```

**"First Working Thing" (Hour 4-6):**
```
Twitter/X:
"Hour [N] at [EventHashtag].

The [feature] just worked for the first time. 

[Screenshot of working product — even if ugly. Especially if ugly.]

Real-time [whatever your product does] in [X] seconds. Not bad. 🛠️"

LinkedIn (same moment, more context):
"[N] hours in at [Hackathon Name].

We hit our first milestone: [feature] is working.

Watching [what the product does] happen in real time is exactly why we chose this problem.

The stack we're using: [2-line description]. [if relevant: using [Sponsor API] for [reason]]

Next challenge: [what's hard right now]

[Screenshot]"
```

**"Mid-Point Honest Update" (Hour 8-12):**
```
Twitter/X:
"Hour [N] update from [EventHashtag]:

✅ [Thing that works]
✅ [Thing that works]  
🔨 [Thing in progress]
❌ Cut [feature] — not enough time

The hard part: [honest challenge]. 

Still here, still building. ⚡"

TIP: The "cut" line shows mature decision-making. Judges notice this.
```

**"Sprint Mode" (Hour 20-22):**
```
Twitter/X:
"[X] hours left at [EventHashtag].

Everything we didn't finish is officially cut.

Right now: demo mode. Making sure the 90-second flow is flawless.

Nervous and excited. 🎯 [EventHashtag]"
```

**"We Shipped" (submission moment):**
```
Twitter/X:
"✅ Submitted.

[Project Name]: [one sentence what it does]

We just spent [X] hours building something we genuinely believe in.

Demo video: [link if available]
GitHub: [link]

Thanks [EventHashtag] — what a ride. 🚀"

LinkedIn (more detailed):
"[Project Name] — submitted at [Hackathon Name] ✅

[2-3 sentences: what it does and why it matters]

The most important thing we learned: [genuine insight]

What we'd do with more time: [1-2 honest things]

Team: [tags]
[Screenshot or demo video]

#hackathon #[EventHashtag] #buildinpublic"
```

## Content Enhancement Tips

### Screenshot Optimization
- **DO:** Screenshot with real data (even seed data) — makes it feel live
- **DO:** Crop to the most impressive single interaction
- **DON'T:** Full-page screenshots (too much empty space, hard to read in feed)
- **DON'T:** Screenshot of terminal/code (unless your audience is technical — and even then, show the output not the code)

### GIF/Video Clips (highest engagement)
If you have 3 extra minutes:
- Record a 10-15 second screen recording of the core interaction
- Upload directly to Twitter (no YouTube link — native video gets more reach)
- Trim to the single most impressive moment

### Hashtag Strategy
```
Primary (always use):
  #[ExactEventHashtag]

Secondary (pick 2-3 based on domain):
  #hackathon #buildinpublic #opentowork (if job seeking)
  
Domain tags (pick 1):
  #AIdev #healthtech #fintech #edtech #climatetech #proptech #govtech #supplychain
  
Tech tags (pick 1-2 only):
  #nextjs #python #openai #supabase #reactnative
```

### Tagging Protocol
- Tag organizers in your submission post ✅
- Tag sponsors when you mention using their tool ✅
- Tag judges only if you've met them in person (don't cold-tag judges) ❌ → ✅ (if met in person)
- Don't tag competitors ❌

## Sponsor-Specific Social Tips

If you're using a sponsor's API/product:
```
"Thanks [Sponsor] for the [API credits / product] — 
[Product name] uses [Sponsor's feature] to [what it does in your product].
[Sponsor handle] [EventHashtag]"

WHY: Sponsors often retweet these. More visibility. Also signals to their prize judges.
```

## Demo Day Social (while waiting to present)

While waiting for your time slot:
```
Twitter/X:
"In the queue to present at [EventHashtag].

[One sentence what we built]

Live demo in [X] minutes. 

Wish us luck 🤞 [EventHashtag]"

(Post-presentation):
"Just presented [Project] at [EventHashtag].

[Honest 1 sentence reaction to how it went]

[Screenshot from the presentation if possible]"
```
